# 要件定義書

## 概要

この機能は、GeminiCLIを使用してLazygitにAI駆動のコミットメッセージ生成機能を追加します。ファイルがステージングされた状態で、ユーザーがAIアシスタントをトリガーして変更を分析し、適切なコミットメッセージを生成できます。生成されたメッセージは、コミット前にレビューと編集のためにコミットメッセージエディターに事前入力されます。

## 要件

### 要件1

**ユーザーストーリー:** Lazygitを使用する開発者として、ステージングされたファイルがある時にAI駆動のコミットメッセージ生成をトリガーしたい。実際のコードの変更に基づいてインテリジェントなコミットメッセージの提案を得るため。

#### 受け入れ基準

1. Lazygitで1つ以上のファイルがステージングされている状態で、ファイルペインで指定されたショートカットキーを押した時、システムは「生成AIがコミットメッセージを作成します。よろしいですか？」という確認ダイアログを表示する
2. 確認ダイアログが表示された時、システムはEscapeキーでキャンセルするか、Enterキーで続行することを許可する
3. 確認ダイアログでEscapeキーを押した時、システムはダイアログを閉じて、変更なしで通常のLazygitインターフェースに戻る
4. 確認ダイアログでEnterキーを押した時、システムはGeminiCLIを呼び出してステージングされた変更を分析し、コミットメッセージを生成する

### 要件2

**ユーザーストーリー:** 開発者として、AIに私のステージングされた変更を分析してもらい、文脈に適したコミットメッセージを生成してもらいたい。コミットメッセージを一から手動で書く必要がないように。

#### 受け入れ基準

1. コミットメッセージを生成するためにGeminiCLIが呼び出された時、システムはステージングされたファイルの変更（diff）をGeminiCLIへの入力として渡す
2. GeminiCLIが変更を処理する時、従来のコミット形式に従い、実際に行われた変更を説明するコミットメッセージを生成する
3. GeminiCLIが正常に完了した時、システムは生成されたコミットメッセージテキストを取得する
4. GeminiCLIが失敗またはエラーを返した場合、システムはユーザーにエラーメッセージを表示し、通常のコミットフローにフォールバックする

### 要件3

**ユーザーストーリー:** 開発者として、生成されたコミットメッセージがコミットエディターに事前入力されることを望む。コミット前にメッセージをレビュー、編集、承認できるように。

#### 受け入れ基準

1. GeminiCLIによってコミットメッセージが正常に生成された時、システムは生成されたメッセージが事前入力されたコミットメッセージエディターを開く
2. コミットメッセージエディターが開いた時、必要に応じて生成されたメッセージを編集できる
3. コミットメッセージエディターを保存して閉じた時、システムは最終的なメッセージを使用して通常のコミットプロセスを続行する
4. コミットメッセージエディターをキャンセルまたはクリアした時、システムは通常通りコミットプロセスを中止する

### 要件4

**ユーザーストーリー:** 開発者として、プラグインがLazygitの既存のインターフェースとシームレスに統合されることを望む。アプリケーションの自然な部分のように感じられるように。

#### 受け入れ基準

1. プラグインがインストールされた時、ファイルがステージングされている時にファイルペインで動作する新しいキーバインディングを登録する
2. ファイルがステージングされていない状態でショートカットキーを押した時、システムは最初にファイルをステージングする必要があることを示すメッセージを表示する
3. AI生成プロセスが実行中の時、システムはローディングインジケーターまたは進行状況メッセージを表示する
4. プロセス中にエラーが発生した時、システムは日本語でユーザーフレンドリーなエラーメッセージを表示する

### 要件5

**ユーザーストーリー:** 開発者として、プラグインが様々なエッジケースを適切に処理することを望む。異なるシナリオで確実に動作するように。

#### 受け入れ基準

1. GeminiCLIがインストールされていないかアクセスできない時、システムはGeminiCLIが必要であることを示すエラーメッセージを表示する
2. 非常に大きなステージングされた変更がある時、システムはAPIの制限を避けるために入力を適切に処理する（必要に応じて切り詰める）
3. 生成されたコミットメッセージが空または無効な時、システムは空のコミットメッセージエディターを開くことにフォールバックする
4. 複数のユーザーが同時に機能を使用しようとした時、各インスタンスは競合なしに独立して動作する
